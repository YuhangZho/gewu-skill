# 残局收尾

## 一句话定位
> 它属于高阶收尾，是用来解决「牌局剩几手时怎么把概率游戏变成数学证明题、确保必胜或反败为胜」的。

## 🎯 核心收获 · 重点知识
**一句话成果**：学完你现在能用「三大场景 + 三原则 + 五步推演 + 10 种经典残局解法」四层框架，把残局从「凭感觉」打成「可证明的必胜题」。

**重点结论**（只收 AI 与你达成的最终共识，3–7 条；筛掉反复出现、一开始没做好后被推翻的）：
- 残局核心：牌权 = 一切决策围绕「我能不能拿到牌权、出完最后手牌」展开。
- 关键转变：残局是斗地主从「概率游戏」变成「数学证明题」的临界点——信息基本明朗，可以推演到终局。
- 必胜公式：控牌 + 跑路组合 = 王 / 2 / 对 2 / 炸弹（控牌）+ 一手牌型（跑路）。
- 三大典型收官场景：一手走完（最简）/ 两手走完（拆控牌 + 跑路）/ 控牌 + 跑路（标准）。
- 三原则：能一手走不拖 / 留控牌 / 为队友清路（农民视角）。
- 五步推演：判信息 → 排场景 → 算牌权 → 验终局 → 择最优。
- 反败为胜三招：逼拆（破坏对手完美结构）+ 诱导（虚招 / 藏牌 / 假双 / 伪装主跑）+ 控节拍（节奏控制决定信息密度）。

## 📝 轻笔记
<details>
<summary>📝 轻笔记（点开看推演摘要与卡壳细节）</summary>

**视觉模型（五问 · 用户先讲）**：
- **对象**：自己剩余手牌、对手剩余手牌（张数 + 可能结构）、牌权、控牌资源（王/2/炸）。
- **连接**：控牌资源 ↔ 牌权（控牌换牌权）；跑路牌型 ↔ 牌权（拿到牌权后一手出完）。
- **流向**：判信息 → 排场景 → 算牌权 → 验终局 → 择最优（五步线性推演）。
- **边界**：信息不充分时不能强行「证明」，需先用诱导招补信息；对手剩 1–2 张时切换逼拆 / 转牌权逻辑。
- **变化**：场景随剩余手数变化（1 手 / 2 手 / 多手），不同场景对应不同控牌 + 跑路组合。

**视觉模型结构（AI 校准后）**：
- **chart_type（AI 自主判定，用户不输入）**：flow
- **节点 kind**：concept（牌权 / 必胜公式）、process（五步推演）、state（三大场景）、rule（三原则）、boundary（信息不充分）
- **结构纠偏**：用户原想把「反败为胜三招」画成独立分支，AI 校准为「五步推演失败时的兜底路径」——只有当推演显示无必胜解时才进入反败为胜模式。

**类比**：它像「下棋残局」，因为剩牌少、信息密度高，每一步都可以推演到终局；区别是斗地主残局还有「诱导 / 藏牌」等不完美信息博弈成分，比纯棋类残局多一层心理战。

**大白话讲解（讲给外行）**：打到剩几张牌时，斗地主就从「猜牌」变成「算题」了。你剩下能出完的牌叫「跑路牌」，你手里能压住对手的牌（王、2、炸弹）叫「控牌」。残局必胜的公式就是：留一张控牌压住对手、剩下是能一手出完的跑路牌。推演时按五步走：先判断已知信息（对手剩几张、出过什么），再排可能的场景，算清谁能拿到牌权，验证到最后能不能赢，最后选最优解。如果算出来赢不了，就用三招反败为胜：逼对手拆牌、用虚招骗他、控制出牌节奏让他出错。记住残局最忌讳「能一手出完偏拆两手」——每多拆一手就多给对手一次翻盘机会。

**三个应用例子**：
1. 你剩 5 张：王炸 + 单 7 + 对 9。对手剩 3 张。按必胜公式控牌（王炸）+ 跑路（单 7 + 对 9 = 两手）→ 不满足「一手跑路」，需调整：先出对 9 试探，对手若压则王炸收尾后单 7 走，若不压则继续单 7 → 王炸留作保险。
2. 你剩 4 张：单 2 + 顺子 34567。对手剩 6 张且未出过 2。按「能一手走不拖」原则直接出顺子，留单 2 作控牌——若对手出 A 你用 2 压回再走顺子；若对手不压则顺子先行留 2 兜底。
3. 你是农民，地主剩 2 张（推测对子），队友剩 1 张大牌。按「为队友清路」原则：拿到牌权出单张逼地主拆对，即便牺牲自己牌型也要让队友先跑——这是农民视角残局收尾的关键。

**失效边界**：这个概念在什么情况下不适用 / 会出错？
- 信息不充分时强行套「必胜公式」会误判——必须先用诱导招补信息（出小牌试探对手结构）。
- 对手剩 1–2 张时不能按常规推演，需切换到「逼拆 / 转牌权」特殊模式。
- 农民视角下「为队友清路」与地主视角「自己一手跑完」逻辑相反，混淆会直接崩盘。

**边界补丁**（失败边界 → 最小补救动作；只演练最容易翻车的 1 个）：
- 边界 1（信息不充分套公式误判）→ 补救动作：五步推演第一步「判信息」若发现对手剩牌结构不明，必须先用一轮小牌诱导——出最小单张或最小对子观察对手是否压、用什么压，补齐信息后再推演。
- 边界 2（对手剩 1–2 张还按常规推演）→ 补救动作：对手剩 1 张直接出非单张转牌权；对手剩 2 张出单逼拆——放弃「一手跑完」优先级，改为「不让对手舒服出完」优先。
- 本轮演练：你剩 4 张（单 2 + 三带对 888+33），地主剩 2 张结构不明。错误做法是直接出三带对赌对手接不上；正确做法是先出单 2 之外的引诱牌（如先出 33 对子），观察地主是否压——若地主压则暴露是否带炸，再决定三带对是否安全出。

**卡壳点复盘**：学的过程中哪里卡住、哪里曾经含糊带过？补上后的正确理解是什么？
- 卡壳点：「控牌」一开始被理解为「所有大牌都是控牌」，结果发现顺子里的 2 也被算进去导致跑路牌型破坏。
- 正确理解：「控牌」特指能压住对手任意牌型的「绝对牌权」——单 2、对 2、王、炸弹；顺子 / 三带里的高牌不算控牌，因为它们不能独立压任意牌型。

**成长对比**：起点认知「残局就是看谁牌大」→ 现在的理解「残局是数学证明题 + 不完美信息博弈，控牌 + 跑路 + 五步推演 + 反败为胜三招才是完整框架」。
