# 算牌与概率推断

## 一句话定位
> 它属于〔斗地主·算牌记牌〕，是用来解决〔如何把记到的牌转化为对手手牌结构推断和概率判断〕的——记牌是数据收集，算牌是数据解读，核心是排除法。

## 🎯 核心收获 · 重点知识
**一句话成果**：学完你现在能用"三层递进推理体系"在 3 秒内完成断张炸弹排查 → 牌型结构反推 → 剩余牌力极限计算，并用"三因素快速评估法"心算断张概率和炸弹分布。

**重点结论**（只收 AI 与你达成的最终共识）：
- 三层递进推理体系：①断张炸弹排查（防守底线，记缺牌推炸）→ ②牌型结构反推（中局核心，从出牌行为推顺子/三带归属）→ ③剩余牌力极限计算（残局决胜，算最大牌归属）。
- 记牌=数据收集，算牌=数据解读，核心是排除法——把自己手牌+已出牌从 54 张里减掉，剩下的就是对手手牌集合。
- 六种行为信号解读：首发选牌（出单牌=单牌多或控单）、跟牌速度（快速跟=早有准备）、不要（断张或藏炸）、拆牌（被迫拆=手数失控）、出牌节奏（突然变慢=关键决策）、犹豫（手中有大牌在权衡）。
- 34 法则行牌推理：自己无 3、下家出 3 → 上家极可能 333（三带）。这是断张+出牌行为联合推理的典型。
- 推理置信度管理：高置信（多源交叉验证）→ 按推断行动；中置信（单一信号）→ 以防万一；低置信（仅推测）→ 仅参考。
- 验证 → 修正闭环：每出一手牌都验证或修正之前的推断，形成贝叶斯更新。
- 关键概率数据（CSDN 实测+理论）：王炸固定概率 ≈ 32%（地主 20 张时）；断 1 张 ≈ 18%、断 3 张 ≈ 50%、断 5 张 ≈ 75%（断张成炸概率随断张数上升）；无王缺 1 门 56% / 缺 3 门 67%；至少 2 人有炸弹 ≈ 12%。
- 三因素快速评估法：数断张 → 看牌型（已出牌结构）→ 观牌型（3 秒心算炸弹分布）。
- 类比"破案排除嫌疑人"+"天气预报降水概率"——排除法减嫌疑，概率论定行动。

**重点知识大纲 / 脑图**：
- 三层递进推理
  - 断张炸弹排查（底线）
  - 牌型结构反推（中局）
  - 剩余牌力计算（残局）
- 排除法核心
  - 54 张 - 自己 - 已出 = 对手
  - 记牌 = 数据收集
  - 算牌 = 数据解读
- 六种行为信号
  - 首发/跟牌速度
  - 不要/拆牌
  - 节奏/犹豫
- 34 法则
  - 自己无 3 + 下家出 3
  - → 上家 333
- 置信度管理
  - 高 → 行动
  - 中 → 防万一
  - 低 → 参考
- 关键概率数据
  - 王炸 ≈ 32%
  - 断张成炸 18%→75%
  - 至少 2 人有炸 ≈ 12%

## 📝 轻笔记
<details>
<summary>📝 轻笔记（点开看推演摘要与卡壳细节）</summary>

**视觉模型（五问 · 用户先讲）**：
- **对象**：54 张牌、自己手牌、已出牌、对手手牌集合、行为信号、概率
- **连接**：断张 → 推炸；行为信号 → 推牌型；概率 → 行动决策
- **流向**：记牌收集 → 三层递进推理 → 置信度评估 → 行动 → 验证修正
- **边界**：低置信时不能 all-in 推断；脑力有限时只跑第一层；概率是期望不是确定
- **变化**：每出一手牌，对手手牌集合缩小，置信度上升

**视觉模型结构（AI 校准后）**：
- **chart_type**：flow
- **节点 kind**：process（推理流程）/ data（概率数据）/ rule（置信规则）/ concept（推理体系）
- **结构纠偏**：用户原以为"算牌=猜对手有什么牌"，AI 校准为"算牌=用排除法确定对手手牌集合，再用概率论量化不确定性"。

**类比**：它像破案排除嫌疑人 + 天气预报降水概率——先把不可能的嫌疑排除（排除法减对手手牌集合），再对剩下的可能性算概率（如断张成炸 18%→75%）。侦探不猜凶手，侦探用证据+概率锁定。

**大白话讲解（讲给外行）**：算牌=把记到的牌转化为推断。三层递进：①断张炸弹排查——你手里没 3，4 张 3 在对手手里，可能成炸，这是防守底线；②牌型结构反推——下家出 3 你没 3，上家大概率 333 三带；③残局算最大牌归属。核心是排除法：54 张 - 你的牌 - 已出牌 = 对手的牌。还有 6 种行为信号：对手出单牌=单牌多或控单；跟牌快=早有准备；不要=断张或藏炸；拆牌=手数失控；突然变慢=关键决策；犹豫=有大牌在权衡。关键概率：王炸 32%、断张成炸随断张数从 18% 升到 75%、至少 2 人有炸 12%。这些数字让你从"感觉"升级到"概率判断"。

**三个应用例子**：
1. **断张炸弹排查**：自己无 4，出牌过半 4 还没出——大概率 4444 在某一家手里成炸。出大牌前先诱炸。
2. **34 法则推理**：自己无 3、下家先手出 3，上家极可能 333（三带）。若 4 也已出完，则下家必无带 3 之顺，可确定上家 333。
3. **行为信号 + 概率联合**：对手过牌且未跟炸（信号=藏炸）+ 自己断 5 张（成炸概率 75%）→ 高置信对手有炸，先诱出再出大牌。

**失效边界**：低置信时不能 all-in（只作参考）；脑力超载时只跑第一层（断张排查）；概率是期望不是确定（32% 王炸≠一定有，只是 1/3 局会有）。

**边界补丁**（失败边界 → 最小补救动作；只演练最容易翻车的 1 个）：
- 边界 1（低置信 all-in）→ 补救动作：低置信只作参考，主力决策仍按手牌结构。
- 边界 2（脑力超载）→ 补救动作：降级到只跑第一层断张排查，其余靠手牌强度判断。
- 本轮演练：边界 1——模拟只有一个行为信号（对手犹豫），不 all-in 推断藏炸，仅作参考保留大牌。

**卡壳点复盘**：一开始把算牌当"猜牌"——猜对手有什么牌，结果经常猜错。后来才理解算牌是"排除法+概率"——先排除不可能的，再对剩余可能性算概率。补上后理解：算牌是数据解读，不是直觉猜牌。

**成长对比**：起点认知"算牌=猜对手手牌" → 现在的理解"算牌=三层递进推理 + 排除法 + 概率论 + 置信度管理 + 验证修正闭环"。
