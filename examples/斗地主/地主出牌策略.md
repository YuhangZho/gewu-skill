# 地主出牌策略

## 一句话定位
> 它属于角色策略，是用来解决「地主一人对两人怎么控场、抢主动、定节奏、稳收尾」的。

## 🎯 核心收获 · 重点知识
**一句话成果**：学完你现在能以「压下放上 + 控牌权 + 藏弱点 + 收尾」四位一体框架打通地主整局，把出牌权死死攥在手里不轻易交给农民接力。

**重点结论**（只收 AI 与你达成的最终共识，3–7 条；筛掉反复出现、一开始没做好后被推翻的）：
- 地主唯一目标 = 不把出牌权轻易交给两个农民；农民一旦接力起手，地主胜率断崖式下跌。
- 核心心法「打下不上」：下家必须全程按住（数据：胜率 +35%），上家不浪费大牌去压，避免两头消耗。
- 三阶段打法：首发选牌（先长后短 + 先繁后简 + 探虚实 + 藏弱点）→ 中局控牌权（忍让控盘 + 80% 赢局靠大牌用在刀刃上）→ 残局收尾（判信息 → 控节奏 → 择最小代价）。
- 强中弱三档分流：强牌强攻（王 + 炸 + 大牌齐全→速战速决）/ 中牌控场（中等控制 + 不错牌型→稳扎稳打）/ 弱牌藏拙（散牌多 + 断张多→绕路走 + 让对手误判）。
- 残局收尾公式：剩 5 张留「炸 + 单」组合；剩 3 张留「对 + 单」组合。
- 失效边界：剩 2–3 张时心法反转——不再「压下放上」，改为「抢牌权第一」。
- 类比「一个人打两个的擂台赛」：体力（牌力）有限，必须用节奏和站位（出牌顺序）放大每一张大牌的价值。

**重点知识大纲 / 脑图**（用缩进嵌套＝树状脑图，一眼看清骨架；对照结构用 Markdown 表格，不单独做 `chart_type`）：
- 目标层
  - 不让出牌权落入农民接力
  - 把大牌用在刀刃上
- 心法层
  - 打下不上（数据 +35% 胜率）
  - 藏弱点 + 探虚实
- 阶段层
  - 首发：先长后短 + 先繁后简
  - 中局：忍让控盘 + 牌权守门
  - 残局：判信息 → 控节奏 → 择最小代价
- 牌力分流
  - 强牌强攻
  - 中牌控场
  - 弱牌藏拙
- 边界与反转
  - 剩 5 张：炸 + 单
  - 剩 3 张：对 + 单
  - 剩 2–3 张：抢牌权第一

## 📝 轻笔记
<details>
<summary>📝 轻笔记（点开看推演摘要与卡壳细节）</summary>

**视觉模型（五问 · 用户先讲）**：
- **对象**：地主、下家农民、上家农民、牌权、残局手数。
- **连接**：地主 ↔ 下家（强压制）；地主 ↔ 上家（弱消耗）；地主 ↔ 牌权（核心争夺对象）。
- **流向**：首发选牌 → 中局控牌权 → 残局收尾，三阶段线性递进。
- **边界**：剩 2–3 张时心法反转；弱牌档不能再硬打「压下放上」。
- **变化**：牌力强中弱三档决定打法路径，残局张数决定是否反转心法。

**视觉模型结构（AI 校准后）**：
- **chart_type（AI 自主判定，用户不输入）**：flow
- **节点 kind**：process（首发/中局/残局）、rule（打下不上）、boundary（张数反转）、state（强中弱档）
- **结构纠偏**：用户原想把「强中弱」画成线性三段，AI 校准为「分流三档」——同一阶段并列三条路径，不是先后关系。

**类比**：它像「一个人打两个的擂台赛」，因为体力（牌力）有限，必须靠节奏和站位（出牌顺序）放大每一张大牌的价值，不能让两个人轮番消耗你。

**大白话讲解（讲给外行）**：你一个人打两个人，最怕的就是被两个人轮流出牌、你插不上手。所以你出牌的规矩就是：紧挨你后面那个人（下家），他出啥你就压啥，绝对不让他舒服地出小牌；离你远那个人（上家），你别浪费大牌去压他，让他出，反正他出完一轮还会回到下家那里。等你手里牌强了，一口气出完；牌弱就装，让对手猜错你的牌。最后剩几张时要换思路——别再想着压下家，谁能抢到出牌权谁赢。

**三个应用例子**：
1. 开局地主手牌：王 + 2 + 顺子 + 三带对 + 散牌。首发直接出长牌顺子（先长后短），既压缩手数又藏住单牌弱点。
2. 中局下家出小单张 4，你手里有 A、K、Q。直接用 A 顶死（打下不上），不让下家继续过小牌；上家出 J 你不要，省下 K/Q 给后面。
3. 残局你剩 5 张：王炸 + 单 7。先出单 7 试探，下家若上 A 你直接王炸收尾——符合「剩 5 张留炸 + 单」公式。

**失效边界**：这个概念在什么情况下不适用 / 会出错？
- 剩 2–3 张时心法反转，若仍坚持「压下不上」会被对手抢先出完。
- 弱牌档强行「压下不上」会耗光手牌控制力，应当转入藏拙绕路。
- 队友接力思路不适用——地主没有队友，所有「配合」逻辑都是错的。

**边界补丁**（失败边界 → 最小补救动作；只演练最容易翻车的 1 个）：
- 边界 1（剩 2–3 张仍按「压下不上」打）→ 补救动作：残局张数 ≤ 3 时切换为「抢牌权第一」——只要能上牌就上，宁可用大牌换出牌权，不留给对手先手出完的机会。
- 边界 2（弱牌档硬打）→ 补救动作：先评估手牌断张数，断张 ≥ 3 张直接转藏拙流——首发故意出对手会要的牌型，引导对手出牌方向，让对手误判你牌力。
- 本轮演练：剩 3 张（对 5 + 单 3），下家出单 4。错误打法是「不压」（按打下不上），正确打法是用单 3 不行→拆对 5 出单 5 抢牌权，再回到对 5 收尾。

**卡壳点复盘**：学的过程中哪里卡住、哪里曾经含糊带过？补上后的正确理解是什么？
- 卡壳点：「压下不上」一开始被理解成「下家出啥都压」，结果发现弱牌档这么打会崩。
- 正确理解：「压下不上」是心法不是铁律，前提是你有压牌资本；牌力不够时优先藏拙，待中局拿到信息再决定是否切回「压下不上」。

**成长对比**：起点认知「地主就是大牌往出甩」→ 现在的理解「地主是节奏战，控牌权 + 阶段分流 + 边界反转才是胜负手」。

</details>
