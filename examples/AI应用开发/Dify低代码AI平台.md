# Dify低代码AI平台

## 一句话定位
> 它属于AI应用开发，是用来解决〔低代码可视化快速搭建 AI 应用〕的——适合快速验证和内部工具场景。

## 🎯 核心收获 · 重点知识

**一句话成果**：学完你现在能判断什么场景该用 Dify（快速验证/内部工具）什么场景该写代码（定制/生产），并用 Dify 搭一个基础 RAG/Agent 应用。

**重点结论**（5 条）：
1. **Dify = 开源低代码 AI 应用平台**：可视化拖拽搭 Workflow/Prompt/RAG/Agent，不用写代码就能出 demo。
2. **核心能力**：Prompt 管理 + RAG 知识库 + Workflow 编排 + Agent + 模型管理（多供应商）+ API 一键发布。
3. **适合场景**：快速原型验证、内部工具、非工程团队自助搭建、教学演示。
4. **不适合场景**：高度定制逻辑、性能敏感、复杂状态管理、需要深度集成的生产系统——这些用 LangChain/LangGraph 写代码。
5. **和 LangChain 关系**：Dify 是"给非开发者用"，LangChain 是"给开发者用"；Dify 底层也用类似编排思想但封装更深。

## 📝 轻笔记
<details>
<summary>📝 轻笔记（点开看推演摘要）</summary>

**起点认知**：以为 Dify 是"玩具"，不知道它在快速验证和内部工具场景的价值。

**核心要点**：Dify 降低了 AI 应用搭建门槛——产品经理/运营都能拖拽搭一个 RAG 问答或 Agent。代价是灵活性受限，复杂逻辑还是得写代码。生产级定制用 LangChain，快速验证用 Dify。

**类比**：Dify 像 AI 应用版的 WordPress/Wix——拖拽建站快，但要做高度定制的电商 SaaS 还得自己写。

**应用例子**：
1. 内部知识库问答：上传文档→配 Prompt→发布，半天出 demo。
2. 客服 Agent：拖 Workflow 配工具，不用写代码。
3. 概念验证：见投资人前用 Dify 2 小时搭原型。
