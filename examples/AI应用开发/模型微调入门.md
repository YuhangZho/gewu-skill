# 模型微调入门

## 一句话定位
> 它属于AI应用开发，是用来解决〔SFT/LoRA 等轻量微调原理与适用场景〕的——应用工程师"了解"层，不必深入训练。

## 🎯 核心收获 · 重点知识

**一句话成果**：学完你现在能判断"这个任务该不该微调、用哪种微调、用什么数据"，面试时讲得清微调和应用层优化的区别。

**重点结论**（6 条）：
1. **微调 = 在预训练模型上用领域数据继续训练**：让模型适应特定任务/领域/风格。区别于 RAG（外挂知识），微调改的是模型参数。
2. **SFT（Supervised Fine-Tuning）**：用"输入-输出"对监督训练，最基础。适合任务/风格定制（如客服语气、特定格式输出）。
3. **LoRA（Low-Rank Adaptation）**：不训全部参数，只训低秩适配矩阵（几 MB vs 几 GB），省显存省时间，效果接近全参微调。是应用层微调事实标准。
4. **QLoRA**：LoRA + 4bit 量化，单卡 24G 显存能微调 70B 模型，进一步降低门槛。
5. **微调 vs RAG 选型**：知识更新/可溯引用 → RAG；任务/风格/格式定制 → 微调；两者可叠加（微调提能力+RAG 接知识）。
6. **应用工程师定位**：会判断该不该微调+会用工具（LLaMA-Factory/Axolotl）跑微调+会评估，不必自己设计训练算法。

## 📝 轻笔记
<details>
<summary>📝 轻笔记（点开看推演摘要）</summary>

**起点认知**：以为微调就是"再训练一下模型"，不知道 LoRA 这种轻量手段和 RAG 的选型区别。

**核心要点**：应用工程师不用自己训模型，但要会判断：知识问题用 RAG，能力/风格问题用微调，LoRA/QLoRA 让微调平民化。微调要数据（几千条起）+评估（别微调完更差）+部署（微调后模型要托管）。

**类比**：RAG 像给模型配参考书（外挂知识），微调像送模型去培训班（内化能力）。参考书随时换，培训完改不了得再培训。

**应用例子**：
1. 客服语气定制：用 5000 条客服对话 SFT，让模型学客服口吻。
2. 代码补全：用领域代码 LoRA 微调，补全准确率提升。
3. 格式输出：微调让模型稳定输出特定 JSON schema。

</details>

### 1. Edward Hu · LoRA 论文一作
- **身份**：前 Microsoft 研究员
- **与本概念的关系**：LoRA 算法提出者，让微调平民化
- **核心洞见**：用低秩矩阵近似参数更新，能用 1% 参数量达到接近全参微调效果
- **代表作**：*LoRA: Low-Rank Adaptation of LLMs*
- **扩展阅读**：[arXiv 论文](https://arxiv.org/abs/2106.09685)

### 2. Tim Dettmers · QLoRA 作者
- **身份**：华盛顿大学研究员
- **与本概念的关系**：QLoRA 把 LoRA+4bit 量化结合，单卡微调 70B
- **核心洞见**：量化+适配器让大模型微调从数据中心走进个人电脑
- **代表作**：*QLoRA: Efficient Finetuning of Quantized LLMs*
- **扩展阅读**：[arXiv 论文](https://arxiv.org/abs/2305.14314)
